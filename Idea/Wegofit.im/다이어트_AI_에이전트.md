# 💡 위고핏 (Wegofit): 초개인화 다이어트 & 웰니스 코치 AI


> **이름의 의미: "Wegofit" (위고핏)**
> 1. 기적의 비만 치료제 '위고비(Wegovy)'에서 착안한 직관적이고 강력한 네이밍.
> 2. "We go fit" (우리는 피트니스를 향해 간다 / 나와 AI가 함께 건강해진다).
> 3. "Our clothes will fit" (다이어트 성공으로 내 옷의 '핏'이 완벽하게 맞게 된다).

## 📌 개요
단순히 칼로리만 계산해 주는 앱을 넘어, **사용자의 실제 생체/운동 데이터(Wearable)와 식단(Diet)을 교차 분석하여 매일 최적의 피드백을 제공하는 '잔소리꾼' AI 에이전트**. 
사용자가 먼저 입력하기 전에 먼저 말을 걸어 식단을 체크하고, 활동량과 섭취량을 냉정하게 평가해 주는 밀착형 헬스케어 비서.

---

## 🎯 핵심 기능 (Core Features)

### 1. ⌚ 웨어러블 기반 생체/운동 데이터 자동 수집
- **API 연동:** Apple Watch (HealthKit), Garmin, Strava API 등과 연동.
- **수집 데이터:**
  - 일일 총 소비 칼로리 및 운동(활동) 칼로리
  - 운동의 종류, 강도, 누적 운동량
  - **HRV (심박변이도) 및 수면 데이터:** 현재 신체의 피로도, 회복 상태, 활력 지표 체크.
- **활용:** "오늘 HRV 수치가 낮으니 무리한 고강도 운동보다는 가벼운 사이클링이나 걷기를 추천해."

### 2. 🥗 스마트 식단 및 공복(간헐적 단식) 관리
- **식단 타겟팅:** 키토제닉(Keto) 베이스 또는 탄수화물을 약간 조절한 '일상식' 중심의 코칭.
- **공복 시간 제어 (Intermittent Fasting):**
  - 목표 공복 시간 설정 (예: 16:8 단식).
  - 공복 구간 내 허용 가능한 식음료 제안 (물, 블랙커피, 애플사이다비니거 등).
  - 공복이 끝나는 시점에 **첫 끼니(보식)로 혈당을 덜 올리는 메뉴 선제적 제안**.
- **Proactive (선제적) 식단 체크:**
  - 식사가 끝날 즈음(또는 예상 식사 시간 직후) 에이전트가 먼저 메시지를 발송: *"오늘 점심 식사 끝났지? 방금 뭐 먹었어?"*
  - 사용자가 대답(텍스트 또는 사진)하면 해당 식단의 영양소(매크로)를 분석하고 다이어트 효과를 즉각적으로 평가.

### 3. 📊 운동 ✖️ 식단 교차 분석 '팩폭' 피드백
- 수집된 운동 데이터와 입력된 식단 데이터를 종합하여 **일일 다이어트 성적표**를 직설적이고 친근하게 피드백.
- **피드백 예시:**
  - *"오늘 가민(Garmin) 데이터 보니까 사이클링으로 500kcal 태우고 신체 활력(HRV)도 빵빵하게 잘 끌어올렸어! 💪 근데 방금 점심으로 먹은 제육덮밥이랑 달달한 라떼 때문에 오늘 다이어트는 사실상 본전치기(유지)야. 🥲 저녁은 무조건 탄수화물 빼고 키토식으로 가자!"*
  - *"오늘 활동량도 적은 데다가 어제 수면 부족으로 회복도 안 됐어. 무리하게 운동하지 말고 오늘은 18시간 공복 유지하면서 푹 쉬는 게 다이어트에 훨씬 도움 돼."*

---


### 4. ⚖️ 체중 및 체성분 데이터 트래킹 (Weight & Body Composition)
다이어트의 최종 지표인 '체중' 데이터를 수집하여 섭취/소비 칼로리와의 상관관계를 분석.
- **방법 1. 스마트 체중계 자동 연동 (API):**
  - Garmin Index, Withings 스마트 체중계 또는 Apple HealthKit과 연동된 체중계(인바디 등)의 데이터를 API로 자동 수집. 아침에 체중계에 올라가기만 하면 에이전트가 즉시 변동 폭을 인지.
- **방법 2. 대화형 수동 입력 (Conversational Input):**
  - 기상 직후 또는 지정된 아침 시간에 에이전트가 선제적으로 물어봄. 
  - *"좋은 아침! 오늘 공복 체중은 어때?"* ➡️ *"75.2kg"* ➡️ *"오! 어제 키토식 잘 지키더니 0.3kg 빠졌네! 킵고잉!"*
- **방법 3. 사진 인증 (Vision AI):**
  - 체중계 눈금 사진이나 인바디 결과지 사진을 찍어서 톡방에 올리면, AI가 숫자를 읽어내고(OCR/Vision) 자동으로 기록 및 평가.


### 5. 🩸 혈당(Glucose) 모니터링 연동 (혈당 스파이크 및 공복혈당)
다이어트의 또 다른 핵심 지표인 '인슐린 저항성'을 관리하기 위해 혈당 데이터를 다이어트/키토 식단과 결합.
- **공복혈당 체크 (Fasting Glucose):**
  - 기상 직후 체중과 함께 '공복혈당' 수치를 수동 또는 헬스 앱(Apple Health)에서 가져옴.
  - *"어제 야식으로 치킨 먹더니 오늘 아침 공복혈당이 110이야! 오늘 점심은 인슐린 쉬게 완전 무탄수 키토식으로 가자."*
- **CGM(연속혈당측정기) 연동:**
  - 리브레(Libre)나 덱스콤(Dexcom) 같은 연속혈당측정기와 연결하여 식사 직후 **'혈당 스파이크'** 발생 시 즉각 경고.
  - *"방금 마신 라떼 때문에 혈당 수직 상승 중! 🚨 당장 스쿼트 50개 하거나 10분만 걷고 와!"*

## 🛠️ 시스템 구현 로직 (구상)
1. **Trigger (트리거):** 
   - 스케줄러 기반 (아침 기상 시 브리핑, 식사 시간대 푸시 알림).
   - 웨어러블 API 웹훅 (운동이 하나 종료될 때마다 즉시 분석).
2. **LLM & Function Calling:**
   - 유저가 "김치찌개에 밥 반 공기 먹었어"라고 하면, AI가 함수를 호출해 대략적인 칼로리와 탄단지 비율을 계산.
3. **User Interface (UI):**
   - 카카오톡, 텔레그램, 혹은 오픈클로 내부 에이전트로 띄워놓고 일상 대화하듯 소통.
