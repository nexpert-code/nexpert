# [기획안] NExpert 지식 공유 및 커뮤니티 플랫폼 전환

이 기획안은 기존 티스토리 블로그(`nexpert.tistory.com`)를 히스토리로 보존하면서, 새 도메인(`www.nexpert.net`)을 옵시디언(Obsidian) 기반의 지식 공유 사이트 및 Webex 인증 커뮤니티로 구축하는 방안을 담고 있습니다.

## 1. 시스템 아키텍처 개요

전체 시스템은 **정적 사이트 제너레이터(SSG)**를 통한 지식 공유 파트와 **동적 웹 앱** 기반의 커뮤니티 파트로 구성됩니다.

| 구분 | 기술 스택 | 주요 역할 |
| :--- | :--- | :--- |
| **지식 공유 (Garden)** | **Quartz 4.0** (Open Source) | Obsidian .md 파일을 트리/그래프 뷰가 포함된 웹사이트로 변환 |
| **인프라/배포** | **GitHub + Vercel/Netlify** | 소스 코드 관리 및 자동 배포 (CI/CD) |
| **커뮤니티 (Forum)** | **Next.js + Supabase** | 사용자 글쓰기, DB 저장 및 권한 관리 |
| **인증 (Workflow)** | **Webex Bot API** | 이메일 기반 승인 요청 및 카드보드 메세지 처리 |

---

## 2. 주요 기능 상세

### 2.1 메인 페이지 및 기존 블로그 연결
- **레이아웃**: Modern & Minimalist 디자인 적용.
- **히스토리 링크**: 페이지 상단 네비게이션 또는 푸터에 "Legacy Blog" 버튼을 배치하여 `https://nexpert.tistory.com/`로 연결.
- **랜딩 섹션**: "AI 및 기술 지식 공유"라는 컨셉을 시각화한 대시보드 형태.

### 2.2 Quartz 기반 Digital Garden (지식 공유)
- **추천 오픈소스: [Quartz (v4)](https://quartz.jzhao.xyz/)**
    - **특징**: Obsidian의 하이퍼링크(`[[Wikilinks]]`), 그래프 뷰, 트리 구조를 완벽하게 지원합니다.
    - **자동화**: 사용자가 Obsidian에서 글을 쓰고 Git Push를 하면, GitHub Actions가 이를 감지하여 `www.nexpert.net`에 즉시 반영합니다.
    - **UI**: 왼쪽에는 **파일 트리(Explorer)**, 오른쪽 또는 하단에는 **그래프 뷰(Interaction Graph)**가 기본 제공됩니다.

### 2.3 Webex Bot 연동 커뮤니티 페이지
- **참여 방식**: 누구나 들어올 수 있지만, 글쓰기 버튼 클릭 시 이메일 입력창이 나타납니다.
- **Webex 인증 워크플로우**:
    1. 사용자가 웹 사이트에서 자신의 **Webex 이메일** 입력.
    2. 시스템이 **Webex Bot**을 통해 운영자(본인)에게 **Adaptive Card** 메시지 발송.
        - 카드 구성: "사용자 [Email]이 글쓰기 권한을 요청했습니다. [승인] / [거절]"
    3. 운영자가 Webex 앱에서 **[승인]** 버튼 클릭.
    4. 시스템 DB(Supabase 등)에 해당 이메일의 권한이 활성화됨.
    5. 사용자는 웹사이트에서 즉시 글쓰기 인터페이스가 활성화됨.

## 3. 구축 단계별 로드맵

### 1단계: 지식 공유 사이트 구축 (1주차)
- GitHub 저장소 생성 및 Quartz 설치.
- Obsidian Vault 연동 (특정 폴더만 업로드되도록 필터링 설정).
- `www.nexpert.net` 도메인 연결 및 SSL 설정.

### 2단계: 커뮤니티 및 Webex 봇 개발 (2주차)
- Webex Developer Portal에서 Bot 생성 및 API 토큰 확보.
- 이메일 입력 및 권한 상태 조회를 위한 백엔드 API 구축.
- Webhooks를 통해 Webex의 카드 액션(`attachmentActions`) 수신 로직 구현.
- **커뮤니티 알림 기능**: 내 글에 답변·댓글이 달리거나 멘션(`@`)이 발생하면 Webex Bot이 해당 사용자에게 실시간 DM 알림 발송.
    - Supabase DB Trigger(또는 서버 이벤트 훅)가 새 댓글/답변/멘션을 감지.
    - 백엔드가 Webex Bot API(`/v1/messages`)를 호출하여 작성자에게 DM 전송.
    - 알림 메시지 예시: "💬 [사용자명]님이 회원님의 글 **[글 제목]**에 댓글을 남겼습니다. → [링크]"
    - 멘션 알림 예시: "🔔 [사용자명]님이 댓글에서 회원님을 언급했습니다. → [링크]"

### 3단계: 통합 및 디자인 고도화 (3주차)
- Quartz 가든과 커뮤니티 페이지 사이의 UI 일관성 작업.
- 메인 페이지에 티스토리 링크 및 소개 컨셉 배치.
- 최종 모바일 최적화 및 테스트.

---

## 4. 운영 및 관리 편의성

- **자동 업데이트**: AI가 생성한 .md 파일이나 옵시디언에서 수정한 내용이 코드 한 줄 수정 없이 웹에 자동 게시됩니다.
- **보안**: Webex의 실제 사용자인지 확인하는 과정을 거치므로 스팸 및 무분별한 게시글 작성을 효율적으로 차단할 수 있습니다.

---

> [!TIP]
> **추천 오픈소스 선정 이유:**
> **Quartz**는 현재 Obsidian 사용자들이 가장 선호하는 퍼블리싱 도구로, 그래프 뷰의 성능과 검색 기능이 매우 뛰어납니다. 커뮤니티 기능의 경우 완전한 오픈소스 포럼(Discourse 등)을 쓰기보다, 사용자의 특별한 요구사항(WebexBot 승인)을 반영하기 위해 **Next.js**로 커스텀 페이지를 하나 추가하는 방식이 가장 깔끔합니다.
