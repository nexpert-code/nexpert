# 🚀 위고핏(Wegofit) 개발 준비 가이드

본 문서는 사용자의 생체 데이터와 식단을 교차 분석하는 AI 에이전트 '위고핏' 개발을 위해 필요한 기술적 준비 사항과 데이터 전략을 정리한 문서입니다.

---

## 1. 헬스 데이터를 가져오기 위한 최적의 API

가장 효율적인 방법은 개별 제조사(Apple, Garmin, Fitbit 등)와 직접 연동하는 대신, 이를 하나로 묶어주는 **데이터 어그리게이터(Aggregator) API**를 사용하는 것입니다.

### 추천 API
1.  **Terra API (가장 권장)**: 전 세계 40개 이상의 웨어러블/헬스 데이터 소스(Apple, Garmin, Strava, Oura 등)를 단 하나의 API로 통합 제공합니다. 스타트업이 빠르게 MVP를 만들기에 가장 적합합니다.
2.  **Vital (Junction)**: 웨어러블 데이터뿐만 아니라 '혈액 검사 결과' 등 임상 데이터까지 연동이 필요할 때 유리합니다.
3.  **OS 기반 직접 연동**:
    - **Apple HealthKit**: iOS 앱 개발 시 필수. 가장 방대한 데이터를 보유하지만 로컬(iPhone)에서만 작동하므로 서버로 전송하는 브릿지 로직이 필요합니다.
    - **Google Health Connect**: 안드로이드 생태계의 표준입니다.

---

## 2. 수집해야 하는 필수 헬스 정보

AI가 정교한 피드백을 주기 위해 매일 수집해야 하는 핵심 데이터 항목입니다.

| 분류 | 필수 데이터 항목 | 용도 및 의미 |
| :--- | :--- | :--- |
| **기본 지표** | 체중, 체지방률, 골격근량 | 다이어트의 최종 성적표 및 장기 추세 분석 |
| **신체 상태** | **HRV(심박변이도)**, 수면 점수 | 신체 피로도 및 회복 상태 파악 (운동 강도 추천) |
| **활동/운동** | 일일 총 소비 칼로리, 심박수 구간(Zones) | 실제 소모 에너지 계산 및 다이어트 강도 평가 |
| **대사 지표** | 공복 혈당, 식후 혈당 추이 | 인슐린 저항성 관리 및 식단 피드백 (혈당 스파이크 방지) |

## 3. 사이클 전문가를 위한 AI 코칭 (예시)

자전거를 타는 사용자라면 단순 칼로리 계산을 넘어 **심박(HR)**과 **VO2max**를 활용한 고차원 피드백이 가능합니다.

### 📈 데이터 분석 로직
- **에어로빅 디커플링(Aerobic Decoupling)**: 동일한 파워(또는 속도)를 낼 때 심박이 얼마나 상승하는지 분석합니다. 심박이 과도하게 오르면 "기초 대사 능력이 부족하니 저강도(Zone 2) 훈련이 필요하다"는 미션을 줍니다.
- **VO2max 추세**: 주간/월간 VO2max 변화량을 추적하여 사용자에게 "현재 심폐 지표가 상승 중이니 이번 주에는 고강도 인터벌 미션을 수행해 보자"고 제안합니다.

### 🎯 AI 미션 부여 예시
- **미션**: "오늘 HRV가 매우 높고 VO2max가 정체기야. 30분 동안 Zone 4(고강도) 인터벌 라이딩을 하고, 운동 직후에는 근성장을 위해 단백질 30g을 꼭 섭취해!"
- **로그 분석**: "가민 데이터를 보니 인터벌 성공했네! 땀 흘린 만큼 오늘 저녁은 당지수가 낮은 샐러드에 구운 생선 어때?"

## 4. 학습을 위한 시스템 및 데이터 준비

AI 에이전트가 똑똑해지기 위해서는 단순 데이터 수집을 넘어 **기계 학습을 위한 파이프라인**이 구축되어야 합니다.

### 1) 시스템적 준비 (Architecture)
- **Time-series Database**: 웨어러블 데이터는 초/분 단위의 시계열 데이터입니다. `InfluxDB`나 `TimescaleDB`를 사용하여 대량의 생체 신호를 효율적으로 저장해야 합니다.
- **Event-driven Webhooks**: 사용자가 운동을 마치고 가민/애플워치를 동기화하는 즉시 서버가 알림을 받아 AI 분석을 실행하는 구조가 필요합니다.

### 2) 학습 데이터 준비
- **사용자 레이블링(Labeling)**: 사용자의 주관적인 피로도(RPE, Perceive Exertion)를 주기적으로 물어보고 수집합니다. (예: "방금 운동 힘들었어? 1~10점 중 몇 점?")
- **데이터 상관관계 학습**: '어제 먹은 야식(식단)' -> '오늘 아침 HRV(회복)' -> '오늘 운동 성과' 사이의 인과 관계를 학습하여 개인별 **'다이어트 성공 공식'**을 찾아내는 강화 학습(Reinforcement Learning) 모델 도입을 고려할 수 있습니다.

### 3) LLM 결합
- 수치 데이터(심박, 칼로리)를 LLM(GPT-4, Claude 등)이 이해할 수 있는 **텍스트 리포트**로 변환하는 프롬프트 엔지니어링이 필요합니다. "사용자의 현재 상태는 Z, 과거 대비 X% 향상됨"과 같은 컨텍스트를 제공해야 합니다.
